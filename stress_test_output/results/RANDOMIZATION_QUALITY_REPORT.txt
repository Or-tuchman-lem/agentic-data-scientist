ROOT CAUSE ANALYSIS
================================================================================

WHAT WENT WRONG?

This is NOT a properly randomized controlled trial. The assignment mechanism
appears to be:

HYPOTHESIS 1: Pseudo-randomization with drift
- Assignment uses hashing/modulo based on customer/quote ID
- Customer characteristics (e.g., existing customers) systematically map
  to certain hash ranges
- Premium-based assignment (e.g., high-premium customers → certain variants)

HYPOTHESIS 2: Sequential or adaptive assignment
- Variants deployed/adjusted over time
- Traffic allocation shifted based on early performance signals
- Geographic rollout: Certain states got certain variants first

HYPOTHESIS 3: Unintentional stratification
- Assignment conditioned on some feature (premium tier, customer type)
- Stratification was unbalanced or incomplete

EVIDENCE FOR HYPOTHESIS 1 (Most Likely):
✓ Premium imbalances (6% CV) suggest premium-based assignment
✓ Existing customer imbalances suggest customer-type clustering
✓ State imbalances (χ²=2,704) suggest geographic hashing artifacts
✓ Temporal imbalances suggest gradual deployment

INVESTIGATION NEEDED:
- Review variant assignment code/logic
- Check if assignment uses customer features as seeds
- Verify if traffic allocation changed during experiment
- Check for geographic or temporal deployment patterns

================================================================================
STATISTICAL METHODS FOR ADJUSTMENT
================================================================================

Option 1: ANCOVA (Analysis of Covariance)
------------------------------------------
Fit model: Sales ~ Variant + Age + Income + PetAge + MultiPet + DebitCard +
                    ConnectedUsers + State + Month

Advantages:
  ✓ Simple to implement and interpret
  ✓ Controls for all observed confounds simultaneously
  ✓ Provides adjusted treatment effects

Disadvantages:
  ✗ Assumes linear relationships
  ✗ Requires sufficient data per variant-covariate combination


Option 2: Propensity Score Matching/Weighting
----------------------------------------------
1. Estimate propensity scores: P(Variant | Covariates)
2. Match treated/control units with similar propensity scores
3. Compare outcomes within matched pairs

Advantages:
  ✓ Non-parametric (fewer assumptions)
  ✓ Balances covariates explicitly
  ✓ Intuitive interpretation

Disadvantages:
  ✗ Can't match all units (lose sample size)
  ✗ Sensitive to model specification
  ✗ Only controls for observed confounds


Option 3: Regression with Interaction Terms
--------------------------------------------
Fit separate models for different customer segments:
- Model 1: Existing customers only
- Model 2: New customers only
- Then pool estimates with meta-analysis

Advantages:
  ✓ Allows heterogeneous treatment effects
  ✓ Interpretable segment-specific recommendations

Disadvantages:
  ✗ Requires large samples per segment
  ✗ More complex to implement


RECOMMENDED APPROACH: Hybrid
-----------------------------
1. Fit ANCOVA model with all covariates (quick, robust)
2. Validate with propensity score weighting (check robustness)
3. Segment analysis for existing vs new customers (interpretability)
4. Compare all three: If consistent → high confidence
                      If divergent → investigate further
