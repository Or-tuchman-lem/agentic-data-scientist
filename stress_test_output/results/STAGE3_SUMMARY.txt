EXECUTIVE SUMMARY

This analysis provides a comprehensive exploration of all 15 predictive features
and their relationships with customer conversion and premium outcomes. Key findings:

🎯 TOP INSIGHTS:

1. **EXISTING CUSTOMER INDICATORS DOMINATE**
   - HAS_MULTIPLE_PET_DISCOUNT: 60.67% vs 14.01% conversion (46.7pp gap!)
   - HAS_DEBIT_CARD: 24.53% vs 14.21% conversion (10.3pp gap)
   - Combined effect: Customers with both convert at 57-64%!

2. **INCOME IS THE STRONGEST NUMERICAL PREDICTOR**
   - Cohen's d = 0.270 (largest numerical effect size)
   - Q4 (highest income) converts at 19.9% vs Q1 at 12.3%
   - Effect varies significantly by state (CA Q4: 22.2%, TX Q4: 16.5%)

3. **PRICE SENSITIVITY CONFIRMED**
   - Lower premiums → Higher conversion (Cohen's d = -0.191)
   - Customers who convert pay $65.62 less on average
   - But: HIGH-VALUE CUSTOMERS convert despite higher prices

4. **GRAPH DATABASE FEATURE WORKS**
   - HAS_STRONGLY_CONNECTED_USERS: 16.64% vs 14.94% conversion
   - Small but statistically significant effect (p=0.0016)
   - Cramér's V = 0.013 (modest effect size)

5. **GEOGRAPHIC VARIATION IS SUBSTANTIAL**
   - DC: 24.87% conversion (highest)
   - HI: 1.61% conversion (lowest)
   - 23.3 percentage point range across states

RECOMMENDATIONS FOR FEATURE ENGINEERING & MODELING
================================================================================

1. FEATURE SELECTION PRIORITIES
   ✓ Must Include:
     - HAS_MULTIPLE_PET_DISCOUNT (strongest predictor)
     - MEDIAN_HOUSEHOLD_INCOME_2020 (strongest numerical)
     - HAS_DEBIT_CARD (existing customer signal)
     - STATE (geographic variation)
     - BASE_PREMIUM (price sensitivity)

   ✓ Strong Candidates:
     - PET_BREED_CLEAN (after grouping rare breeds)
     - IMPUTED_AGE
     - PET_AGE_YEARS
     - HAS_STRONGLY_CONNECTED_USERS (graph DB feature)
     - TOTAL_VET_CLINICS

   ⚠ Consider Excluding:
     - PIT_ANNUAL_PREMIUM (redundant with BASE_PREMIUM, r=0.998)
     - COVERAGE_SENT_AT (timestamp, high cardinality)
     - QUOTE_MONTH_COS (weak signal)
     - PET_SEX (negligible effect)

2. FEATURE ENGINEERING OPPORTUNITIES

   A. Income Transformations
      • Create income quartiles/quintiles (non-linear effect detected)
      • Log transformation to handle skewness
      • Income × State interactions

   B. Breed Grouping
      • Group rare breeds (n < 50) into "other"
      • Create breed categories: popular (top 20), common (21-100), rare (101+)
      • Consider breed size/species aggregation

   C. Age Transformations
      • Customer age bins: 18-25, 26-35, 36-45, 46-60, 60+
      • Pet age bins: puppy/kitten (0-1), young (2-3), adult (4-7), senior (8+)
      • Age interactions with pricing

   D. Geographic Features
      • State-level aggregations (region: West, South, Midwest, Northeast)
      • Income rank within state (percentile)
      • Vet clinic density quartiles

   E. Composite Features
      • Existing customer score: HAS_MULTIPLE_PET_DISCOUNT + HAS_DEBIT_CARD + HAS_STRONGLY_CONNECTED_USERS
      • Value score: Income quartile + Premium level
      • Engagement score: Debit card + Connected users

3. MODELING RECOMMENDATIONS

   A. Handle Class Imbalance
      • Only 15.08% conversion rate
      • Consider: SMOTE, class weights, stratified sampling
      • Optimize for precision-recall, not just accuracy

   B. Model Architecture
      • Tree-based models (XGBoost, LightGBM) will handle:
        - Non-linear relationships
        - Interactions automatically
        - Mixed data types
        - Outliers robustly
      • Consider neural networks if interaction complexity is high

   C. Feature Interactions to Explore
      • Multi-pet discount × Debit card (already confirmed)
      • Income × State (already confirmed)
      • Age × Premium
      • Breed × State
      • Connected users × Income

   D. Model Evaluation
      • Primary metric: Average sales per quote ($)
      • Secondary: Conversion rate, average premium
      • Use holdout test set with temporal split (last month)
      • Validate on state-level subgroups

4. DATA QUALITY NOTES

================================================================================
KEY TAKEAWAYS FOR NEXT STAGE (FEATURE ENGINEERING)
================================================================================

1. **Prioritize Existing Customer Signals**
   - Multi-pet discount effect is massive (60% vs 14%)
   - Debit card and graph DB features add incremental value
   - Create composite "existing customer" score

2. **Income is Key, But Context Matters**
   - Strongest numerical predictor (Cohen's d = 0.270)
   - Effect varies by state (interaction present)
   - Non-linear relationship (quartiles show clear steps)

3. **Price Sensitivity is Real but Modest**
   - Lower premiums increase conversion slightly
   - But high-value customers convert despite high prices
   - Segment-specific pricing strategies may be optimal

4. **Geographic and Demographic Segmentation Required**
   - State-level variation is large (1.6% to 24.9%)
   - Age and pet age show consistent negative effects
   - Consider state-specific or region-specific models

5. **Feature Engineering Will Be Critical**
   - Raw features show modest effects
   - Interactions and transformations will unlock more signal
   - Breed grouping, income binning, age categories recommended
